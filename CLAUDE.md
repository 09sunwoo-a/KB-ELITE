# 펀드 길잡이 에이전트 (KB-ELITE)

고객의 막연한 언어를 상품 조건으로 번역해 펀드 후보를 발견하게 돕는 개인화 AI 탐색 에이전트.
LangGraph 백엔드 + Streamlit 시연 UI + 노션 발표자료가 최종 산출물이다.

## 문서가 계약이다 — 작업 전 필독 순서

1. `01_펀드_길잡이_서비스_컨셉_대화플로우_v4.md` — 서비스 정체성, 노출 규칙(4-2), 페르소나·대본
2. `02_langgraph_architecture.md` — 그래프 6노드, State, 도구, 가드레일
3. `03_rag_data_design.md` — 데이터 파이프라인, funds.json 스키마, 검증 assert
4. `04_streamlit_demo_ui.md` — UI 구조, **UI↔Agent 데이터 계약(§6)**
5. `05_stage_plan_and_acceptance.md` — 단계별 게이트, 수용 기준

구현이 문서와 어긋나면 코드로 우회하지 말고 문서를 먼저 고친 뒤 구현을 맞춘다.
`AI_펀드_길잡이_Streamlit_시연화면_기획안.md`는 04의 원본 초안일 뿐이다 — 충돌 시 04가 우선.

## 절대 원칙 (서비스 정체성)

- **탐색이지 추천이 아니다**: "추천드립니다/적합합니다/가입하세요/더 낫습니다" 금지. 복수 후보 + 근거 + 차이 + 선택권.
- **투자성향 = 노출 상한 하드 필터** (01 4-2): 안정형2 / 안정추구3 / 위험중립4 / 적극5 / 공격6. 성향 초과 상품은 후보로 노출하지 않되, 차단 사실·사유·대안을 안내한다.
- **수치는 LLM이 아니라 funds.json·계산 도구에서만** 나온다. 데이터에 없는 수치(종목 비중, 글라이드패스, 분배 내역)는 표시·계산하지 않고 한계를 고지한다.
- 페르소나는 테스트 데이터일 뿐 — 런타임이 페르소나 ID·연령으로 분기하지 않는다.

## 스테이지 게이트 (05 §5)

- 단계 산출물 완성 → 확인 방법 실행 결과를 사용자에게 보여주고 **멈춘다** → 사용자가 통과 선언 후 다음 단계.
- 단계를 건너뛰거나 여러 단계를 몰아서 완료 처리하지 않는다.
- 게이트 통과 시 커밋 1회: `A1: data pipeline — gate passed` 형식.

## 병렬 세션 소유권 (05 §5-4)

| 트랙 | 소유 (쓰기 가능) | 금지 |
|---|---|---|
| A 백엔드 | `build_data.py`, `data/`, `app/`, `tests/` | `ui/`, `assets/` 수정 금지 |
| B 프론트 | `ui/`, `assets/`, 발표자료 | `app/`, `data/`, `build_data.py` 수정 금지 (읽기·import만) |

트랙 B는 백엔드 완성 전에는 `MockAgentAdapter`(04 §7)로 개발하고 `MOCK TRACE` 배지를 표시한다.

## 기술 스택·규약

- Python + LangGraph, Anthropic Claude(라우터 `claude-sonnet-5` / 응답·칩 생성 `claude-haiku-4-5`), 임베딩은 OpenAI `text-embedding-3-small` 유지, `InMemoryVectorStore`, `InMemorySaver` (02 §9 채택표 준수 — ReAct·멀티에이전트·Store·HITL 도입 금지)
- API 키는 `.env`만 사용, 하드코딩·커밋 금지
- `펀드정보.xlsx`(311건)는 원본 — 수정 금지. 파생 데이터는 `build_data.py`로만 생성

## 1. 컨셉 — 검색기가 아니라 ‘발견의 대화’

**고객의 막연한 마음에서 출발해, 고객이 말한 목적과 조건을 상품 데이터로 번역하고, 선택 가능한 펀드 후보와 차이를 발견하도록 돕는 개인화 AI 탐색 에이전트.**

고객은 항상 원하는 펀드를 정확히 알고 들어오지 않는다. “돈을 좀 굴려볼까”, “예금만 두긴 아까운데” 같은 막연한 마음으로 들어오기도 한다. 에이전트는 다음 순서로 탐색을 돕는다.

1. **듣는다** — 고객의 말을 곧바로 상품 조건으로 단정하지 않고, 이번 돈의 목적과 사용 시점 등 필요한 맥락을 확인한다.
2. **되짚어준다** — 고객의 말을 탐색 가능한 기준으로 번역해 보여주고, 해석이 맞는지 확인한다.
3. **후보를 발견하게 한다** — 대화에서 확인한 조건을 충족하는 복수 후보를 근거와 함께 제시하고, 차이를 비교할 수 있게 한다.

성공 장면은 고객이 “어, 이거였네”라고 말하는 순간이다.

> **“이 상품을 선택하세요”가 아니라 “말씀하신 조건으로 찾으면 이런 후보가 있고, 차이는 이렇습니다.”**  
> 에이전트는 고객 대신 상품을 판단하지 않는다. 고객이 판단할 수 있도록 조건·근거·차이를 구조화한다.

## 2. 문제 정의

기존 모바일 펀드 화면은 정보와 필터가 풍부하지만 **고객 주도 탐색형 UI**다. 고객이 직접 기준을 만들고, 용어를 해석하고, 필터링하고, 여러 상세 화면을 오가며 비교해야 한다.

1. 처음 접하는 고객은 무엇을 기준으로 봐야 할지 모른다.
2. 용어가 어려운 고객은 학습과 상품 탐색이 서로 분리되어 있다.
3. 조건이 있는 고객도 자연어를 상품 필드(비용·위험등급·지역·보유종목 등)로 직접 변환해야 한다.
4. 고객의 이해 수준과 보유상황이 달라도 같은 목록과 같은 설명이 제공된다.

**문제의 본질은 상품 정보의 부재가 아니라, 고객의 언어·목적·상황과 상품 데이터 사이의 탐색 공백이다.**

## 3. 설계 원칙

1. **탐색이지, 추천이 아니다**  
   “적합합니다”, “추천드립니다”, “가입하세요”, “이 상품이 더 낫습니다”와 같은 판단형 표현을 사용하지 않는다. 복수 후보, 선정 근거, 차이, 선택권을 제공한다.

2. **개인화는 판단이 아니라 맥락에 쓴다**  
   고객 데이터는 질문을 줄이고, 설명 눈높이를 조정하고, 비교 기준선을 제공하는 데 사용한다. 개인화를 이유로 후보를 숨기거나 한 상품으로 좁히지 않는다. 단, 투자성향에 따른 노출 차단(4-2)은 개인화 판단이 아니라 모든 고객에게 동일한 기준으로 적용되는 규제 기반 안전 규칙이다.

3. **투자성향과 이번 자금의 조건을 구분한다**  
   공식 투자성향은 가입 가능한 위험등급의 상한을 정하는 노출 규칙(4-2)이다. 이번에 투자할 돈의 기간·사용 가능성·손실 감내 수준은 그 범위 안에서 후보를 좁히는 탐색 조건으로 별도 확인하며, 성향으로부터 이번 자금의 조건을 추정하지 않는다.

4. **모든 사실에는 근거와 기준일이 있다**  
   상품 수치와 사실은 RAG 원문 또는 정형 데이터에서만 가져온다. 데이터가 없거나 공개 범위가 제한된 경우 계산하지 않고 한계를 밝힌다.

5. **페르소나는 구현 분기 기준이 아니다**  
   페르소나는 시연과 테스트를 위한 데이터 세트다. 런타임 로직이 페르소나 ID, 연령, 성별만으로 설명 수준이나 검색 결과를 결정해서는 안 된다. 발화의 구체성, 세션에서 확인한 목적, 이해 반응, 명시된 응답 선호를 기준으로 행동을 선택한다.

## 4. 개인화 설계

### 4-1. 고객 데이터 스키마

투자성향 진단은 사전에 완료된 것으로 가정한다. 다만 공식 투자성향과 이번 자금의 탐색조건은 분리한다.

  "risk_profile": "안정추구형",

    "goal": null,
    "horizon_years": null,
    "liquidity_need": null,
    "loss_tolerance": null,

  "account_context": {

#### 필드 사용 원칙

- `risk_profile`: 노출 상한 산출(4-2)과 차단 발생 시 안내 문구 삽입에 사용한다.
- `investment_context`: 이번 세션에서 질문하거나 고객 발화에서 추출해 갱신한다.
- `interaction_preference`: 명시된 선호 또는 세션 반응으로 갱신한다. 연령으로 추정하지 않는다.
- `holdings`: 고객 동의 및 조회 권한이 있는 범위에서 비교 기준선으로만 사용한다.
- `search_history`: 첫 화면의 빠른 시작 칩 소재에만 약하게 반영하며 검색 랭킹에는 직접 가중하지 않는다.

### 4-2. 투자성향 노출 규칙 — 하드 필터

고객의 공식 투자성향(5단계)은 가입 가능한 펀드 위험등급(6단계)의 상한을 정한다. 상한을 초과하는 펀드는 검색·비교 후보로 노출하지 않는다.

| 투자성향 | 노출 상한 risk_score | 가입(노출) 가능 위험등급 |
|---|---|---|
| 안정형 | 2 | 매우낮은위험, 낮은위험 |
| 안정추구형 | 3 | + 보통위험 |
| 위험중립형 | 4 | + 다소높은위험 |
| 적극투자형 | 5 | + 높은위험 |
| 공격투자형 | 6 | 전체 (매우높은위험 포함) |

운영 원칙:

1. 이 규칙은 개인화가 아니다. 모든 고객에게 동일한 기준으로 적용되는 규제 기반 안전 규칙이며, 검색 도구가 코드로 보장한다(02 6장).
2. 조건에 맞는 상품이 성향 초과로 제외된 경우, 제외된 사실과 사유를 안내한다. 차단하되, 차단했다는 사실은 숨기지 않는다.
3. 고객이 성향 초과 상품·테마를 명시적으로 요청해도 후보로 제시하지 않고, 차단 사유와 함께 범위 내 대안 또는 이전 탐색으로 되돌아갈 선택지를 제공한다.
4. 고객에게는 위험등급 텍스트만 노출하고 risk_score 숫자는 내부 계산용으로만 쓴다.

### 4-3. 개인화 적용 7지점과 우선순위

| # | 적용 지점 | 구현 방식 | 스코프 |
|---|---|---|---|
| ① | 오프닝 문장 + 빠른 시작 칩 3개 | 프로필과 최근 관심 주제를 이용해 능력 소개의 강조점과 칩 소재를 변주 | **필수** |
| ② | 질문 축소 | 세션에서 이미 확인된 목적·기간·납입방식은 다시 묻지 않음 | **필수** |
| ③ | 설명 눈높이 | 발화의 전문성, 이전 설명 반응, 명시된 응답 선호를 State에 반영 | **필수** |
| ④ | 성향 초과 노출 차단·안내 | 검색 도구가 노출 규칙(4-2)을 코드로 적용, 차단 발생 시 후처리로 안내 문구 삽입 | **필수** |
| ⑤ | 매 턴 후속 칩 | 탐색 노트와 현재 행동을 바탕으로 다음 가능한 행동 3개 제시 | **하이라이트** |
| ⑥ | 보유상품·보유종목 겹침 분석 | 보유 데이터와 펀드 구성 데이터를 동일 기준일·식별자로 매칭해 계산 | **하이라이트** |
| ⑦ | 검색 히스토리 검색 가중치 | 검색 품질 왜곡 가능성이 있어 제외. ①⑤의 칩 소재에만 사용 | 제외 |

①~⑤는 프로필과 탐색 노트를 State에 로드해 템플릿·프롬프트·후처리에 주입하는 동일한 메커니즘으로 구현한다. ⑥은 별도의 정형 계산 도구를 사용한다.

**개인화의 3단 체감 구조**: 첫 화면에서 보이고 → 불필요한 질문이 줄어드는 것으로 느껴지고 → 비교·겹침 분석에서 증명된다.

### 4-4. 은은한 개인화 원칙

기본은 **은은하게**다. 개인화는 설명 방식, 질문 생략, 칩 구성, 비교 기준선에 반영하되 대화 중 개인 데이터를 불필요하게 복창하지 않는다.

- ❌ 행동 단정: “다시 오셨네요”, “지난번에 보셨던”, “관심 있으셨죠?”
- ❌ 민감한 개인 데이터 복창: 투자성향 명칭, 보유상품명, 투자금액을 먼저 말하기
- ❌ 보유상품 평가: “요즘 많이 떨어졌죠”와 같은 평가
- ❌ 개인화를 이유로 후보를 한 개로 축소하거나 임의로 가리기 (투자성향 노출 규칙 4-2는 개인화가 아니라 전 고객 동일 적용 규칙)
- ❌ 연령만으로 이해 수준, 은퇴연도, 위험 선호를 단정하기

보유상품 비교 기능처럼 개인 데이터가 직접 등장하는 경우에는 먼저 비교 범위를 설명하고 고객의 선택을 받는다.

> “현재 보유 중인 상품을 기준으로 주요 종목과 자산군의 중복을 비교해 볼까요?”

### 4-5. 기억 범위

이번 스코프는 **세션 내 개인화만** 구현한다. 과거 검색어는 mock 프로필의 정적 데이터로 제공한다. 세션 종료 시 탐색 노트는 폐기한다.

- 현재 버전: 세션 내 State + Checkpointer
- 확장 버전: 고객 동의 기반 탐색 노트 Store 저장 및 “이어보기”

### 4-6. 계산 가능 범위와 데이터 조건

#### 비용 원 단위 환산

- `총보수·비용률`, `클래스`, `기준일`이 있는 경우에만 계산한다.
- 목돈은 예상 운용금액 × 연간 비용률로 단순 환산한다.
- 적립식은 총 납입액에 단순 곱하지 않고, 납입 시점별 예상 평균잔액을 사용한다.
- 실제 비용은 기준가격과 납입일에 따라 달라질 수 있음을 함께 표시한다.

#### 보유종목 겹침 분석

- 데이터셋에 표준 종목코드가 없으므로 alias 사전 기반 정규화 매칭(03)을 사용한다.
- 펀드의 공개된 구성종목 범위와 기준일을 함께 표시한다.
- 데이터셋의 주요보유종목에는 종목별 비중이 없다. 따라서 겹침 분석은 “공개 상위 보유종목 기준 겹치는 종목명”까지만 제공하고, 펀드 내 비중과 전체 포트폴리오 중복률은 계산하지 않으며 그 한계를 함께 고지한다.

#### TDF 비교

- 숫자만으로 고객에게 특정 빈티지를 제시하지 않는다.
- 예상 은퇴시점, 계좌 편입 가능 여부(적격 태그), 빈티지, 위험등급, 비용, 유형을 기준으로 비교한다.
- 데이터셋에 자산배분·글라이드패스 수치가 없으므로, 글라이드패스는 일반 개념 설명(용어 사전)으로만 다루고 상품별 수치 비교는 하지 않는다.

#### 월지급식

- 데이터셋에 분배 주기·분배 내역 데이터가 없다. 후보는 인컴·월지급 태그 기반으로 제시하고, 분배 구조는 일반 개념으로 설명하되 개별 상품의 분배 내역·지급액은 제공 불가를 고지한다.

## 5. 대화 구조 — 탐색 노트 중심 턴 루프

### 5-1. 세션 흐름

```text
[세션 시작] 프로필 로드 → 탐색 노트 초기화 → 개인화 오프닝과 빠른 시작 칩 생성
[턴 루프]  ① 발화 이해: 의도·조건·질문 추출
          ② 탐색 노트 갱신: 목적·기간·손실 감내·후보·설명한 용어 누적
          ③ 행동 선택: 설명 / 질문 / 검색 / 비교
          ④ 도구 실행: RAG 검색 / 정형 조회 / 계산
          ⑤ 응답 생성: 근거 + 기준일 + 안전 후처리 + 후속 칩 3개
[세션 종료] 탐색 노트 폐기
```

### 5-2. 탐색 노트

| 영역 | 필드 | 초기값 |
|---|---|---|
| 안전 기준선 | 공식 투자성향, 노출 상한(risk_score) | 프로필에서 로드, 4-2 매핑으로 산출 |
| 고객 컨텍스트 | 보유 요약, 계좌 유형, 납입방식, 명시된 응답 선호 | 프로필에서 로드 |
| 이번 자금 조건 | 목적, 사용 예정 시점, 중도 사용 가능성, 손실 감내 수준 | 미정, 대화에서 갱신 |
| 상품 조건 | 지역·테마, 관심 종목, 비용 민감도, 상품 유형, 분배 방식 | 미정 |
| 탐색 이력 | 본 후보 상품코드, 설명한 용어, 마지막 검색 조건 | 빈 값 |
| 대화 제어 | 직전 행동, 연속 질문 횟수, 마지막 근거 데이터 기준일 | 빈 값 |

### 5-3. 행동 4종

| 행동 | 트리거 | 응답 형태 |
|---|---|---|
| 설명 `explain` | 용어·구조·위험에 대한 질문 | 핵심 설명 + 오개념 교정 + 탐색으로 잇는 선택지 |
| 질문 `ask` | 현재 의도에 필요한 필수 맥락이 부족함 | 질문 1개 + 선택지 예시 |
| 검색 `search` | 구체 조건이 있거나 현재 의도의 필수정보가 확보됨 | 후보 2~4개 + 선정 기준 + 핵심 데이터 + 후속 칩 |
| 비교 `compare` | 후보 2개 이상을 특정하거나 비교를 요청함 | 공통 기준 비교표 + 필요 시 보유 기준선 |

### 5-4. 행동 선택 규칙

우선순위는 단순 고정 순서가 아니라 **사용자 의도와 현재 후보 상태**로 결정한다.

1. 명시적인 용어 질문이면 먼저 설명한다.
2. 특정 후보 간 비교 요청이면 비교한다.
3. 구체적인 상품 조건이 충분하면 질문 없이 검색한다.
4. 막연한 탐색이면 현재 의도에 필요한 핵심 맥락만 질문한다.
5. 연속 질문은 최대 2턴으로 제한하고, 그 이후에는 확보된 조건으로 넓게 검색해 후보를 보며 좁힌다.

#### 의도별 필수정보 예시

| 의도 | 검색 전에 필요한 정보 |
|---|---|
| 특정 종목·테마 펀드 | 종목 또는 테마가 명시되면 즉시 검색 가능 |
| 비용 낮은 상품 | 상품 범주 또는 지역이 없으면 한 번 확인 |
| 막연한 여유자금 운용 | 사용 시점과 손실 감내 수준 확인 |
| 연금·TDF | 계좌 유형, 예상 목표 시점 또는 비교 희망 빈티지 확인 |
| 월지급식 | 분배 구조 설명 후 위험 수준 또는 자산 유형 확인 |

검색 가능 여부는 “슬롯 두 개 충족” 같은 일률적 개수로 판단하지 않는다.

### 5-5. 빠른 시작 칩 운영 원칙

UI 명칭은 `추천 질문` 대신 **빠른 시작** 또는 **이렇게 물어보세요**를 사용한다.

첫 화면의 칩 3개는 가능하면 다음 역할을 하나씩 포함한다.

1. **이해하기** — 용어·위험·상품 구조 질문
2. **찾아보기** — 조건 또는 목적 기반 탐색
3. **비교하기** — 후보나 보유상품과의 차이 확인

칩은 과거 행동을 직접 언급하지 않고 현재 사용할 수 있는 기능처럼 표현한다.

- ❌ `지난번에 본 TDF 이어보기`
- ❌ `친구가 말한 S&P500이 뭔가요?`
- ✅ `TDF 숫자의 의미 알아보기`
- ✅ `매달 나눠 넣는 상품 살펴보기`
- ✅ `보유상품과 주요 종목 비교하기`

### 7-3. 교차 시연 카드 — 같은 질문, 다른 결과: 노출 규칙의 투명한 적용

동일한 질문을 두 페르소나에게 입력한다.

> “요즘 AI 펀드 어때요?”

#### 최준혁 (공격투자형 — 전 위험등급 노출 가능)

- 질문의 대상이 명확하므로 즉시 후보 제시 (매우높은위험 포함 전 상품이 노출 범위)
- 상품별 위험등급, 비용, 손실 가능성 정보는 기본 표시 — 정보는 모든 고객에게 동일

#### 서지우 (안정추구형 — 보통위험까지 노출 가능)

- 조회된 AI 펀드가 모두 노출 상한(3) 초과 → 후보를 제시하지 않음
- 차단 사실과 사유(전부 높은위험 이상, 성향 범위 초과)를 명확히 안내
- 범위 내 대안 탐색 또는 이전 후보로 돌아갈 선택지 제공

**내레이션**

> 같은 질문이라도 투자성향 노출 규칙에 따라 결과가 달라집니다. 이는 개인화 판단이 아니라 모든 고객에게 동일하게 적용되는 규칙이며, 에이전트는 차단을 숨기지 않고 사유와 대안을 함께 안내합니다.

## 8. 톤 가이드 & 고객 언어 사전

### 8-1. 오프닝 공식

**인사 → 에이전트 능력 소개 → 시작 제안 → 빠른 시작 칩 3개**

개인화는 능력 소개의 강조점과 칩 소재에만 은은하게 드러낸다.

### 8-2. 말투 규칙

- `OOO 고객님` 호칭과 정중한 해요체를 사용한다.
- 친절하지만 과도하게 친근하거나 감정적으로 부추기지 않는다.
- “좋은 타이밍이에요”, “시간이 고객님 편이에요”처럼 투자 시점이나 회복 가능성을 낙관적으로 단정하지 않는다.
- “맞는 상품”, “더 나은 상품”, “보완 상품” 같은 판단형 표현을 피한다.
- 한 응답에는 핵심 설명 1개와 다음 행동 선택지를 제공한다.
- 데이터에 없는 수치와 미래 수익을 생성하지 않는다.
- 상품 사실에는 가능한 경우 기준일을 표시한다.

### 8-3. 고객 언어 사전

| 내부 용어 | 고객 언어 표현 |
|---|---|
| 총보수·비용 | 매년 펀드 자산에서 빠져나가는 보수와 비용 |
| 매우낮은위험·낮은위험 | 다른 펀드보다 가격 변동이 상대적으로 작은 편 |
| 높은위험 | 가격 변동과 손실 가능성이 큰 편 |
| 분배금 | 정해진 시기에 지급될 수 있는 돈 |
| 주요보유종목 | 이 펀드가 담고 있는 주요 종목 |
| 적립식 | 매달 나눠 넣기 |
| 거치식 | 목돈 한 번에 넣기 |
| 변동성 | 가격이 오르내리는 정도 |
| 글라이드패스 | 목표 시점에 가까워지면서 자산 비중을 조정하는 방식 |
| 성향 초과 | 진단받은 투자성향 범위보다 위험 수준이 높은 상품 |

고객 언어 표현도 사실을 지나치게 단순화해서는 안 된다.

- ❌ 낮은위험 → `원금이 거의 줄지 않는`
- ❌ 분배금 → `매달 받는 이자`
- ❌ 높은위험 → `오르는 만큼 내리는 상품`
- ✅ 낮은위험 → `상대적으로 가격 변동이 작은 편이지만 손실 가능성은 있음`
- ✅ 분배금 → `지급액이 변할 수 있고 지급 후 기준가격이 낮아질 수 있는 돈`

역방향 고객언어 → 상품조건 매핑은 `03_rag_data_design.md`에서 정의한다.

### 8-4. 안전장치 요약

- 투자권유·수익보장 표현 제거: 프롬프트 + 금칙어·패턴 후처리
- 성향 초과 노출 차단: 검색 도구의 코드 필터(4-2) + 차단 발생 시 안내 문구 삽입
- 수치와 상품 사실: 정형 데이터 조회 우선
- 비용·겹침 계산: 코드 도구 사용, 계산 근거와 기준일 표시
- 데이터 불충분 시 계산 중단 및 범위 고지
- 최근 수익률: 반드시 기준기간과 기준일 표시, 미래 성과 암시 금지
